How I build

A product manager's process, an engineer's execution

Every build runs through the same disciplined loop — I scope it like a PM, architect it like an engineer, and use AI agents as my dev team. Security, auth, and data protection are designed in from phase one, not bolted on.

01 · DISCOVER

Problem & users

Define the real problem, the user, and the single outcome that means it works. Write the spec before any code.

02 · DESIGN

UX & architecture

Flows, data model, auth model, and the integration map. Decide the stack and where secrets live.

03 · BUILD

Agent-driven sprints

Task-sliced, iterative sprints. AI agents write code against a clear spec; I review, test, and integrate.

04 · HARDEN

Auth, keys, data

Sessions, RBAC, row-level security, encrypted keys, mock-mode fallbacks, and compliance guardrails.

05 · SHIP

Deploy & iterate

One-click deploy (Railway / Vercel), env config, seed data, and a reproducible run guide.
